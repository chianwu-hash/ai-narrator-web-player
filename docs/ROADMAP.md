# 功能待辦紀錄

最後更新：2026-07-17

本文件記錄播放器已完成、暫不做與後續可排程的功能。它不是法律授權文件，只是開發與管理維護用的工作紀錄。

## 已完成

### 1. 許願池

狀態：已完成 MVP。

已包含：

- 前台「許願池」入口。
- 使用者匿名送出想聽的書、作者與推薦理由。
- 前台公開顯示其他人的許願內容。
- SVG 水池視覺。
- 以 Open Library 查找可用封面，將封面做成書本卡片放入池中。
- 後台 `/admin/wishes` 管理願望狀態。
- 不要求使用者記名，也不記錄使用者聽了什麼書。

參考文件：

- `docs/WISH_POOL.md`
- `supabase/migrations/202607170004_book_wishes.sql`

### 2. 書籍 / 單集留言

狀態：已完成 MVP，且需求已從「只給管理者看」調整為「前台公開」。

已包含：

- 書籍留言。
- 單集留言。
- 留言類型：感想、重大錯誤回報、其他。
- 前台所有使用者可看到留言。
- 同一瀏覽器可編輯或刪除自己的留言。
- 管理者可在 `/admin/comments` 調整狀態或刪除留言。
- 後台不記錄播放進度，不追蹤使用者聽了什麼書。

參考文件：

- `docs/ADMIN_COMMENTS.md`
- `supabase/migrations/202607170002_content_comments.sql`
- `supabase/migrations/202607170003_comment_ownership.sql`

### 3. 播放器風格自選

狀態：已完成 MVP。

已包含：

- 使用者可自行選擇播放器風格。
- 初始風格：書房綠、紙頁暖、夜讀墨。
- 風格偏好保存在本機。
- 若使用者已開啟跨設備同步，`themeId` 會跟著同步。
- 風格不影響播放進度、最愛、留言或許願池。

### 4. 跨設備同步

狀態：已完成 MVP。

已包含：

- 匿名收聽檔案。
- 第一台設備可開啟同步。
- 已同步設備可產生配對碼。
- 新設備或新瀏覽器輸入配對碼後同步播放進度、最愛、倍速、最後播放集數與風格。
- 不把共用邀請碼當成個人進度身份。

參考文件：

- `docs/SUPABASE_SYNC.md`
- `supabase/migrations/202607170001_sync.sql`

### 5. 設備使用監控

狀態：已完成 MVP，正在觀察實際資料。

已包含：

- 前台登入後定期送出匿名設備活動。
- 後台 `/admin/devices` 顯示總設備數、24 小時活躍、7 天活躍與匿名設備列表。
- 不記錄書名、單集、播放進度或使用者姓名。

參考文件：

- `docs/DEVICE_MONITORING.md`
- `supabase/migrations/202607170005_device_activity.sql`

### 6. 後台導覽與總覽

狀態：已完成基礎版。

已包含：

- `/admin` 管理總覽。
- 後台共用導覽列。
- 後台「回播放器」連結。
- 前台「說明」與「管理」入口。
- `/help` 登入後說明頁。
- 總覽顯示新留言、新許願與 24 小時活躍設備數。

## 暫不做

### 公開徵集或投票

許願池只作為私人需求收集，不做公開投票、排名或大規模徵集。

### 記錄使用者聽了什麼書

設備監控只統計匿名設備活動，不記錄書名、單集、播放時間軸或閱讀偏好。

### 使用者帳號系統

目前維持「邀請碼 + 匿名同步檔案 + 配對碼」模式，不建立個人帳號、Email 登入或 OAuth。

### 後台直接修改邀請碼

邀請碼仍由 Vercel 環境變數管理。這比較麻煩，但安全邊界清楚。若未來要做後台更換邀請碼，需要另外設計權限與審計紀錄。

## 建議下一步

### 1. 確認 Supabase migration 都已執行

尤其是設備監控：

- `supabase/migrations/202607170005_device_activity.sql`

若未執行，前台仍可使用，但 `/admin/devices` 與總覽設備統計會沒有資料。

### 2. 實機觀察設備監控

用手機、桌機、不同瀏覽器登入後，觀察 `/admin/devices` 是否出現匿名設備紀錄。

需要確認：

- iOS Safari 是否會送出活動。
- Chrome / Edge 是否分別被視為不同設備。
- 清除網站資料後是否會被視為新設備。

### 3. 同步 Telegram 群組 `/help`

狀態：文案已準備，尚未套用到 bot。原因是目前工作區沒有 Telegram bot 原始碼或 `/help` 指令來源。

標準文案已整理於：

- `docs/TELEGRAM_HELP.md`

取得 Telegram bot repo 或實際 `/help` 檔案後，應同步目前網站說明：

- 網站網址。
- 邀請碼用途。
- 管理者碼只供後台使用。
- 不私自分享邀請碼與內容。
- 管理者會不定期更換邀請碼。
- 本網站旨在個人研究與導讀，不取代原書閱讀。
- NotebookLM 目前 4 個帳號輪值，一天可規劃 4 本書。

### 4. 邀請碼更換紀錄

可先不用做成正式後台功能，但建議以文件方式記錄：

- 更換日期。
- 更換原因。
- 是否已通知親友。
- 是否需要讓舊 session 失效。

### 5. 管理後台的低成本改善

## 長期才考慮

### 音訊流量與成本優化

目前音訊仍經 Vercel API proxy。若使用量提高，應觀察 Vercel Functions 與流量成本，再評估短效簽名 URL、私有 CDN 或其他儲存方案。

### 更完整的管理稽核

若使用者範圍擴大，後台操作可以增加稽核紀錄，例如誰刪了留言、何時更改邀請碼。但這會引入更多管理資料，應確認必要性後再做。
