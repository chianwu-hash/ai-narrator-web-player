# 功能待辦紀錄

本文件記錄已討論但尚未實作的播放器功能。這些功能都應維持「私人、低資料收集、低擴散風險」原則。

## 1. 許願池

### 目的

讓受邀使用者匿名提出「想聽的書」與推薦理由，協助管理者整理後續製作優先順序。

### MVP 範圍

- 前台新增「許願池」頁籤或入口。
- 表單欄位：
  - 想聽的書，必填。
  - 推薦理由，建議必填，但限制字數。
- 不記名。
- 不顯示公開排行榜。
- 不允許上傳檔案。
- 送出後顯示簡短回饋，例如「已收到，管理者會視情況整理」。

### 建議資料表

```text
book_wishes
- id
- title
- reason
- status
- created_at
```

`status` 可先保留給管理者使用：

```text
new
reviewing
accepted
rejected
done
```

### 風險邊界

許願池只做私人需求收集，不做公開徵集、投票或上傳來源檔案。這樣比較不會把播放器變成公開書單徵集平台。

## 2. 書籍 / 單集留言

### 目的

讓使用者針對特定書籍或單集留下：

- 聽後感想。
- 導讀回饋。
- 說書人的重大錯誤回報。

這個功能主要服務品質修正與內容回饋，不用來追蹤使用者閱讀或收聽習慣。

### MVP 範圍

- 書籍頁可留言。
- 單集可留言。
- 留言不記名。
- 留言預設只給管理者看，前台不公開展示。
- 留言類型可先用簡單選項：
  - 感想
  - 錯誤回報
  - 其他
- 錯誤回報可讓使用者描述：
  - 哪一段有問題。
  - 問題原因。
  - 建議修正方向。

### 建議資料表

```text
content_comments
- id
- target_type
- book_id
- episode_id
- comment_type
- body
- created_at
- status
```

`target_type`：

```text
book
episode
```

`comment_type`：

```text
reflection
error_report
other
```

`status`：

```text
new
reviewing
resolved
ignored
```

### 隱私與資料最小化

- 不要求姓名。
- 不記錄使用者聽了哪些書。
- 不公開顯示留言。
- 不把留言和播放進度綁定。
- 若為了防濫用需要設備識別，僅使用匿名 device id 或 hash，不顯示在管理介面。

### 風險邊界

留言功能應定位為私人回饋與品質修正，不做社群討論區。避免公開留言、公開書籍需求或公開錯誤討論造成額外擴散風險。
