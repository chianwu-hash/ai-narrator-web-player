# Telegram 群組 `/help` 文案

最後更新：2026-07-17

目前本機專案中沒有 Telegram bot 程式碼或 `/help` 指令來源；播放器 repo 也在 `README.md` 註明它獨立於既有 Telegram bot。因此本文件先保存最新版 `/help` 標準文案。取得 bot repo 或實際 `/help` 檔案後，直接以本文件內容同步。

## 建議 `/help` 文案

```text
📚 AI 說書人使用說明

播放器網址：
https://ai-narrator-web-player.vercel.app

一、登入與同步

• 邀請碼：用來登入播放器。請勿私自轉傳或公開分享。
• 配對碼：只用來連結新設備或新瀏覽器，讓播放進度、最愛與頁面風格跨設備同步。
• 管理者碼：只供管理者登入後台，一般使用者不需要。

如果換手機、換電腦、換瀏覽器，請先用邀請碼登入播放器，再從已同步設備產生配對碼完成連結。

二、法律與分享提醒

本群組與播放器內容涉及原書、教材或其他著作的重製與導讀，可能有著作權風險。

目前只限親友小範圍使用，並以邀請碼作為最低限度的私人存取控制。請不要私自分享邀請碼、播放器網址、音訊檔或整理內容。

管理者會視情況不定期更換邀請碼；更換後，舊登入狀態可能會失效，需要重新登入。

三、開發宗旨

本網站開發旨在個人研究與導讀，絕不是要取代閱讀。
AI 說書只能協助快速理解與複習；原書閱讀才能體會文字之美。

四、目前功能

• 播放 AI 說書音訊
• 跨設備同步播放進度與最愛
• 書籍與單集留言，可留下感想或指出重大錯誤
• 許願池，可匿名提出想聽的書與推薦理由
• 播放器風格自選

五、NotebookLM 製書配額

目前有 4 個帳號輪值，因此一天最多可規劃 4 本新書；實際可用狀況仍以 NotebookLM 當日限制為準。

若流程暫停，會保留斷點，可用 /book_retry 繼續。
```

## 管理者維護提醒

- 如果播放器網址改變，這份文案與 Telegram `/help` 都要同步更新。
- 如果邀請碼更換，通常不需要把邀請碼明文寫進 `/help`，只需提醒使用者向管理者取得。
- 如果 NotebookLM 帳號數量改變，請同步更新每日可規劃本數。
- 如果未來後台新增邀請碼管理，需重新檢查「管理者碼」與「邀請碼」的說明是否仍正確。

## 套用狀態

尚未直接套用到 Telegram bot，原因：目前工作區沒有 bot 原始碼或部署入口。

需要取得其中之一：

- Telegram bot 的 GitHub repo。
- 本機 bot 專案路徑。
- 實際儲存 `/help` 內容的檔案。
- 若 `/help` 由資料庫或環境變數控制，則需要對應後台或部署設定位置。
